    There are several places in {{ CourSys }} where you can create formatted content, and several options of how to specify the formatting.

    These should all work the same wherever the option is presented…

    WikiCreole

    This options interprets your content as Wiki Creole Markup (list of Wiki Creole markup reference). Most Creole Additions also work.

    Some additions have been made to Creole markup that might be useful for course web pages…

    Due Dates

    You can automatically insert the due date of an activity like this:

    Assignment 1 is due <<duedate Assignment 1>>.

    As long as you spell the activity name correctly, the page will display the due date assigned for the activity and will update if you change it. If you'd like to include the date and the time, it is done like this:

    Assignment 2 is due at //exactly// <<duedatetime Assignment 2>>.

    Page Lists

    You can insert a list of all pages on your course site like this:

    <<pagelist>>

    Or, a list of all pages whose label starts with “Assign” (e.g. pages “Assignment 1”, “Assignment 2”) like this:

    <<pagelist Assign>>

    Hilighted Code

    Code samples can be syntax highlighted if you specify the language of the code in question. Here is an example of Python code that will be highlighted when displayed:

    {{{ [python]
for i in range(10):
    print i
}}}

    Markdown

    The “Markdown” option interprets your content as GitHub Flavoured Markdown.

    In particular, code syntax highlighting work as on GitHub:

    ```python
for i in range(10):
    print i
```

    Textile

    Textile markup is supported as a legacy feature: we suggest you move to Markdown for new content.

    HTML

    A subset of HTML markup is supported directly. Unsafe markup will be stripped for security reasons.

    Code syntax highlighting is supported with <pre lang=""> like this:

    <pre lang="python">
for i in range(10):
    print i
</pre>

    HTML Editor

    The “HTML Editor” option lets you edit HTML markup with a nice point-and-click editor. Restrictions and details are the same as hand-entered HTML, as described above.

    Plain Text

    Self explanatory. There is zero formatting here, everything will show up exactly as you type it, in plain text.

    Formulas and MathJax

    If you enable the “Use MathJax?” option, your content will be processed with MathJax to allow TeX math-mode formulas.

    When integrating MathJax with one of the above markup languages, it can take some effort to escape things properly: backslashes are special in Markdown and carets in Creole. HTML may be the easiest option if you're writing math-heavy content.

    Code Syntax Highlighting

    Most common programming languages should be available as described above. If you need support for one that isn't, ask {{ help_email }}.
